Question 1

For the three categories of web software technology we've just defined, you are asked to find out which category so-called "Netscape plug-in" technology belongs to. Please describe the basic idea of the technology, and give your reasoning on its position in the technology categories.

Category

Distributed Object Technology.

Basic Idea

It can handle files of content-types which the browser can not handle natively. When the browser encounters a file that it can not handle, the plugin will be loaded and be responsible for rendering the data correctly like an external application running in the browser.

Reason

Like Microsoft's ActiveX, plug-ins can be called by the browser to render a certain file. They work together, but they reside in different processes or threads and they can only communicate using a common interface(defined in NPAPI).

Question 2

In a normal intercommunication between Web browser and Web Server, Web Server closes the network communication right after it sends back the document requested by the browser. Think about this statusless character of World Wide Web. Give some good points and bad points of it. Can a web server know exactly how many browsers are reading its document?

Good Points

  • It can lessen the burden of the Internet server, especially when there are many users are connecting this website.
  • Though the maximum number of active processes is limited, the server can deal with more requests because it needn't be all transmitted at the same time.
  • The status data is far less valuable than the Web page content itself, and in most cases we don't need the status data. So it can save the resources which are used by the web server.

Bad Points

  • The Web server doesn't keep any memories about a certain client, and it is difficult for a server to determine whether two connections come from the same client if other technologies like Cookies are not introduced.
  • For websites with user authentication info, it will be bothersome if we have to log in to the server every time.
